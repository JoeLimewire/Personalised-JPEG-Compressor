What is JPEG Compression?


JPEG is a lossy compressing standard for images all across the world, famous for reducing the file size of images massively while maintaining the image "quality". The ratio of compression is around 10:1, meaning you could fit a 1GB image into around 127MB! This is HUGE when thinking of all the images in the world, how much space that must save!


Why don't you try to reduce an images quality above until you can see differences in both images?
To do this, follow these steps:


○ Select an Image from the green drop down box on the bottom of the application.
○ Play with the slider to get the desired quality from the compression. See the 'quality' value change as you move the slider.
○ Click the quantise button to begin processing.

Hover over the image to see what its done to the pixels



Essentially it grabs blocks of pixels (8x8) and it trys to find certain shapes to recreate the look of the block. It repeats this for all 3 colour values(red, green and blue), then all it has to store is what shapes it uses then it can make the block of pixels again.
Still don't understand?
Hopefully this deeper explanation of the process can help you to understand.

There are 5 nessasary steps to dealing with compression. These steps include:


○ RGB to YCbCr Conversion
○ Splitting and Shifting this data, preparing for compression.
○ Finding DCT Coefficients for each block
○ Quantisation
○ Recomposition


RGB to YCbCr Conversion


YCbCr is another way of representing colour within a pixel. It is split by luminance ([Y], also known as brightness) and Chrominance([Cb] and [Cr] also known as hue). These values can be closely predicted by using the appropriate equations which we apply to each pixel to get the value we need.
